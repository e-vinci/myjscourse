L'essentiel

L’essentiel du contenu de ce cours a été repris dans une série de démonstrations (ou tutoriels) nommée « essentials ».

Tout au long de la série « essentials », un site pour une pizzeria est développé de manière incrémentale.

Chaque démo de la série « essentials » illustre certaines compétences du cours. Le fichier README décrit les étapes importantes dans la création de la démo. Il vous est conseillé de lire ce fichier README en parallèle du code.

Voici quelques explications concernant les grandes étapes du développement d’une Single Page Application (SPA) :

1. Création d’une version préliminaire du frontend

Cette version du frontend reprend tout la mise en page souhaitée (header, footer, main, barre de navigation…) et permet d’afficher les différentes pages qui, généralement, sont juste une version statique de celles-ci. A cette étape, la mise à jour dynamique du frontend via des opérations sur des Web APIs n’est pas encore envisagé. N’hésitez pas néanmoins à cette étape-ci à rendre votre frontend dynamique pour tout ce qui peut être fait côté client (affichages des bons composants lors de clics, animations…).

2. Création de la RESTful API

Cette Web API va mettre à disposition des opérations sur des ressources (les données) nécessaires pour rendre votre site web dynamique.

3. Création de votre Single Page Application (SPA)

Cette étape consiste à mettre à jour votre frontend afin que celui-ci s’intègre avec votre RESTful API. Votre site devient dynamique sur base d’opérations sur des ressources offertes par votre RESTful API.

Nous espérons que cette série vous facilitera le développement de votre propre site.
